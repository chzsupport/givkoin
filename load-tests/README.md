# Локальные нагрузочные проверки GIVKOIN

Эта папка отделяет нагрузочные проверки от основного кода проекта.

## Что здесь есть

- `run-local-load-suite.js` - безопасный локальный прогон через отдельный стенд. Он создает 5000 временных пользователей в памяти процесса, прогоняет стадии `500, 1000, 1500 ... 5000`, собирает задержки, ошибки, память и примерные слабые места.
- `create-local-users.js` - создание 5000 тестовых пользователей в локальной Supabase. Скрипт откажется работать, если база не на `localhost` или `127.0.0.1`.
- `cleanup-local-users.js` - удаление тестовых пользователей после оптимизации. Без явного подтверждения не запускается.
- `results/` - сюда складываются отчеты после прогона.
- `state/` - сюда складывается список созданных тестовых пользователей, чтобы не создавать их заново без причины.

## Быстрый безопасный прогон

```powershell
node load-tests/run-local-load-suite.js --start=500 --step=500 --max=5000 --duration=20
```

Этот вариант не трогает настоящую базу. Пользователи живут только внутри временного локального стенда.

## Реальные локальные пользователи

Сначала нужна локальная Supabase в `backend/.env`:

```env
SUPABASE_URL=http://127.0.0.1:54321
SUPABASE_KEY=...
```

Потом:

```powershell
node load-tests/create-local-users.js --count=5000
```

Скрипт помечает пользователей меткой `givkoin_load_local`. Сразу удалять их не нужно.

## Очистка после оптимизации

Только когда тестовые пользователи больше не нужны:

```powershell
node load-tests/cleanup-local-users.js --confirm=DELETE_LOAD_TEST_USERS
```

Скрипт тоже откажется работать с нелокальной базой.
